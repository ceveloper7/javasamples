Concepto Big O
==============
Describe el ritmo con el que crece la complejidad o el tiempo de ejecucion de un algoritmo.
Lo importante es el ritmo de crecimiento y no el numerode instrucciones ejecutadas.

Big O(n) -> siendo n el numero de elementos.

LA eficiencia de un algoritmo vendra dada por su comportamiento en los extremos. POr el tiempo de ejecucion
o Big O

Tiempos de ejecucion posibles
=============================
Big O(N) -> complejidad lineal "mas rapida que la complejidad constante"
Big O(N^2)
Big O(log N)
Big O(N log N)
Big O(2^N)

Complejidad Algoritmica
=======================
La complejidad algoritmica mide la forma en la que crece el tiempo de ejecucion de un algoritmo segun los valores de entrada

Complejidad temporal -> tiempo que toma ejecutarse un algoritmo
Complejidad Espacial -> cantidad de memoria utilizada en el algoritmo

CONSTANTES
==========
Big O(N) y Big O(5N) -> tienen un mismo ritmo de crecimiento lineal y no importa la constante. Solo el crecimiento exponencial O(2^N) es
                        mayor que el crecimiento linean O(N). O(N) y O(5N) crecen al mismo ritmo y forman la misma figura una linea recta

Big O(2^N) y Big O(5^N) -> poseen el mismo crecimiento exponencial a pesar de tener constantes distintas. Solo la complejidad factorial
                           es superior a la complejidad exponencial.

Complejidades Big O Mas Comunes
===============================
Big O(1) -> es el mejor, la complejidad constante
Big O(Log N) -> La segunda mejor complejidad (aunque estas dos complejidades son dificiles de conseguir en algoritmos complejos)
Big O(N) -> Complejidad Lineal
BIg O(N * LOGN)

-- Malas complejidades
Big O(N^2) -> complejidad cuadratica
Big O(2^N) -> Complejidad exponencial
